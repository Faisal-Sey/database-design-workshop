# Exercise 1 — Pick Your Database

**Time:** 10 minutes (individual or pairs)
**Tools needed:** Pen and paper, or the chat box

---

## The Scenario

You are joining a startup in Accra. The team has three products they want to build and they need you to recommend a database for each one. For each scenario below:

1. Name the database type you would use (relational / document / key-value / time-series / graph)
2. Give one specific product (e.g. "PostgreSQL", "MongoDB", "Redis")
3. Write one sentence explaining your main reason

---

## Scenario A — GhanaPostGPS Address Book

A government app that stores verified addresses for every building in Ghana. Each address has GPS coordinates, a digital address code (e.g. `GA-039-5028`), property type, and links to a district and region. Users search by digital code or GPS proximity.

**Your answer:**
- Database type: ___________
- Specific product: ___________
- Reason: ___________

A mobile app used by lecturers to mark attendance. Each class session records which students were present or absent. Reports are needed at the end of semester: attendance percentage per student, per course.

## Scenario C — Live Football Match Score App

An app that shows live scores for Ghana Premier League matches. Scores update every few seconds during a game. Millions of users refresh during peak match times. Historical scores need to be queryable by date and team.

## Debrief Questions (discuss with the group)

- Did anyone pick different databases for the same scenario? Why?
- Which scenario was hardest to decide? What made it hard?
- For Scenario C — what if the app also needs to store full historical statistics going back 10 years? Does your answer change?
